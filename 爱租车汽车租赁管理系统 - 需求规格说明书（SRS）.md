## 1. 引言

### 1.1 项目背景

随着汽车租赁行业的快速发展，传统的人工管理方式（纸质合同、手工记录车辆状态、Excel统计经营数据等）已无法满足业务增长需求。门店日常运营中面临车辆状态不透明、订单追踪困难、客户信息散落、违章处理滞后等问题，严重影响了服务效率和客户体验。

**爱租车系统**旨在为汽车租赁公司提供一个集**车辆管理、客户管理、订单管理、违章管理、经营统计**于一体的数字化管理平台，帮助门店实现业务流程线上化、车辆状态实时化、经营数据可视化，从而提升运营效率与客户满意度。

### 1.2 项目目标

1. **业务流程线上化**：实现从车辆入库、客户建档、订单创建、取还车到违章处理的完整业务闭环，告别纸质单据和手工台账。
2. **车辆状态实时化**：通过系统实时追踪每辆车的业务状态（可租用/已租出/维修中/已下架），避免人工沟通带来的信息滞后和租车冲突。
3. **经营数据可视化**：通过数据看板直观展示订单量、营收趋势、车辆出租率等核心指标，为经营决策提供数据支撑。
4. **管理规范化**：通过系统配置、操作日志、客户黑名单等功能，规范内部管理流程，降低运营风险。

### 1.3 术语定义

| 术语       | 说明                                                         |
| :--------- | :----------------------------------------------------------- |
| 系统管理员 | 拥有系统全部管理权限的人员，可进行用户管理、系统参数配置及数据维护 |
| 业务操作员 | 负责日常业务办理的普通员工，可操作车辆、客户、订单、违章等业务模块 |
| 客户黑名单 | 因信用问题或违规行为被限制租车的客户管控机制                 |
| 结算凭证   | 还车完成后生成的费用明细单据，包含租金、超时费、押金退还等详细信息 |
| 车辆状态   | 车辆当前所处的业务状态，包括可租用、已租出、维修中、已下架四种情况 |
| 订单状态   | 订单从创建到完成所经历的各阶段状态，反映租车业务的实时进度   |
| 验车记录   | 还车时对车辆外观、内饰、功能等进行检查后留下的图文记录       |
| 超时费用   | 客户未按约定时间归还车辆所产生的额外租赁费用                 |
| 违章处理   | 车辆租赁期间产生的交通违章记录，在确认责任人后进行扣分、罚款缴纳等处理操作 |

### 2.3 数据库配置

```yaml
# application.yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3307/aizuche?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: pwd
  jpa:
    hibernate:
      ddl-auto: update          # 开发环境用 update，生产环境用 validate
    show-sql: true              # 开发环境开启SQL日志，生产环境关闭
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
        format_sql: true
server:
  port: 8081                    # 应用服务端口
```

### 2.4 数据库初始化步骤

1. 安装 MySQL 8.0，确保服务已启动
2. 创建数据库：`CREATE DATABASE aizuche DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
3. 配置 `application.yaml` 中的数据库连接信息
4. 启动应用，JPA 自动建表（`ddl-auto: update`）
5. 首次启动自动创建管理员账号 `admin / 123456`（DataInitializer）

### 4.3 核心用例规约示例（还车处理）

| 项目         | 详细内容    |
| :----------- | :---------- |
| **用例名称** | 还车处理    |
| **所属模块** | 订单管理 |
| **优先级**   | 高          |

**1. 用例概述**

业务操作员在客户归还车辆时，录入实际还车时间、检查车况、生成结算凭证并完成押金退还。

**2. 参与者**

- 主要参与者：业务操作员
- 次要参与者：客户（接收结算凭证）

**3. 前置条件**

1. 订单状态为"已租出"。
2. 车辆已返回门店指定停车点。

**4. 后置条件**

- **成功**：订单状态变更为"已完成"，车辆状态恢复为"可租用"，生成结算凭证。
- **失败**：订单状态不变，记录异常原因。

**5. 基本事件流**

| 步骤 | 参与者动作                       | 系统响应                                                 |
| :--- | :------------------------------- | :------------------------------------------------------- |
| 1    | 业务操作员在还车页面选择当前订单 | 展示订单详情（客户信息、车辆信息、租期、押金金额）       |
| 2    | 录入实际还车时间                 | 系统自动计算超时费用                                     |
| 3    | 录入车况描述并上传验车照片       | 保存验车记录                                             |
| 4    | 点击"生成结算凭证"               | 系统核算最终费用（租金+超时费-优惠）                     |
| 5    | 选择押金退还方式，输入退还金额   | 记录押金退还信息                                         |
| 6    | 点击"确认还车"                   | 更新订单状态为"已完成"，车辆状态为"可租用"，推送结算凭证 |

**6. 业务规则**

- BR-01：超时费用 = 超时时长 × 日租金 × 超时费率（超时费率可在系统配置中设置）。
- BR-02：验车照片至少上传3张（车前、车侧、车后）。
- BR-03：结算凭证一经生成不可修改，如需调整需走冲销流程。

## 8. 开发规范

### 8.1 Java 编码规范

| 规范项 | 要求 |
| :----- | :--- |
| **命名风格** | 类名 UpperCamelCase，方法名/变量名 lowerCamelCase，常量 UPPER_SNAKE_CASE |
| **包结构** | `com.aizuche.{layer}` 分层分包 |
| **注解** | Controller 使用 `@RestController` + `@RequestMapping`；Service 使用 `@Service` + `@Transactional` |
| **注入** | 统一使用 `@Autowired` 字段注入 |
| **日志** | 使用 Slf4j，Controller 中记录关键操作日志 |
| **异常处理** | Controller 返回 `Result.error()` 统一格式，避免直接抛出异常到前端 |

### 8.2 前端编码规范

| 规范项 | 要求 |
| :----- | :--- |
| **框架** | 纯原生 JavaScript（ES2020+），不依赖任何前端框架 |
| **样式** | 使用 css/common.css 设计系统中的 CSS 变量和组件类 |
| **API 调用** | 封装 `$get / $post / $put / $del` 统一处理，响应格式为 `{code, msg, data}` |
| **状态管理** | 使用 `sessionStorage` 存储登录用户信息 |
| **路由** | 单页应用通过 `showPage()` 函数切换视图，非 SPA 页面通过 `window.location` 跳转 |
| **弹窗** | 统一使用 `showConfirm(title, msg, icon, okText, okClass)` 自定义确认弹窗 |
| **通知** | 统一使用 `toast(message, type)` 函数 |

### 8.3 数据库规范

| 规范项 | 要求 |
| :----- | :--- |
| **表命名** | 全部以 `t_` 前缀开头（如 `t_user`, `t_vehicle`） |
| **字段命名** | 使用下划线分隔（`snake_case`），Java 实体使用驼峰映射 |
| **主键** | 统一使用 `BIGINT` 自增ID |
| **时间字段** | `create_time`（创建时间）、`update_time`（更新时间）两张标配 |
| **字符集** | `utf8mb4`，排序规则 `utf8mb4_unicode_ci` |
| **冗余设计** | 订单表冗余存储客户姓名、手机号、车辆品牌型号车牌，减少多表关联查询 |
